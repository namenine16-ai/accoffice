# Sprint 03

## Objective

Implement the monthly workflow module for managing accounting tasks and customer work cycles.

## Completed work

- Added monthly workflow page with summary cards and task table.
- Created filters for month, year, task status, customer, employee, and text search.

- Extended `CustomerTask` data model to capture workflow state, deadlines, assignment, and document status.
- Built reusable workflow components to keep the page maintainable and responsive.

## Acceptance criteria

- Workflow data loads correctly from API filters.
- Users can review task progress and status at a glance.
- The workflow UI is responsive and accessible.
- Build and lint checks pass.
